1. LOYIHA HAQIDA
================================================================
Mone Bakery task boshqaruv ilovasi.
- Backend: Go (Golang) + SQLite
- Frontend: Flutter (Dart)
- Real-time: WebSocket
- State: Provider (ChangeNotifier)

3 ta rol:
- super_admin: Tasklar yaratish, tahrirlash, o'chirish, tekshirish
- checker: Tasklarni ko'rish, video ko'rib tekshirish (50% auto-approve)
- worker: Tasklarni bajarish, video yuborish


2. ARXITEKTURA
================================================================

6. STATUS LOGIKASI
================================================================

Task statuslari:
- null  → hali ko'rilmagan / video yuborilgan, tekshirmagan
- 1     → rad etilgan (qizil)
- 2     → kutilmoqda/qayta ko'rish (sariq)
- 3     → tasdiqlangan (yashil)

Qoidalar:
- Status 3 qo'lda o'zgartirib BO'LMAYDI (na checker, na super_admin)
- Checker 50% video ko'rsa → avtomatik status 3
- Super_admin status 3 qila olmaydi
- Status 1, 2 → qo'lda o'zgartirish mumkin (video bo'lsa)
- Status 1, 2 bosilganda → avval audio yoziladi, keyin status o'zgaradi


7. AUDIO LOGIKASI
================================================================

- Har bir taskda max 2 ta audio saqlash mumkin
- DB da checker_audio_url TEXT fieldida JSON array saqlanadi:
  '["url1.m4a", "url2.m4a"]'
- 3-chi audio yuborilsa eng eskisi (1-chi) o'chiriladi
- Barcha rollar (worker, checker, admin) audio yubora oladi
- Audio o'chirish: DELETE /api/tasks/{id}/voice-comment/{date}/{index}
- Eski format (bitta string) ham o'qiladi (backward compatible)


8. VIDEO LOGIKASI
================================================================

- Worker video yozib yuboradi (segmentli yoki bitta)
- Video yuborilganda status NULL ga o'tadi
- Segmentli videolar backgroundda FFmpeg bilan birlashtiriladi
- Video cache: VideoDownloadProvider orqali local cache
- Video player: CircleVideoPlayer (PageView + arc progress)


9. WEBSOCKET
================================================================

Flutter: WsService (singleton, auto-reconnect 3s)
Backend: gorilla/websocket, Hub pattern

Flow:
1. Client JWT token bilan /ws/tasks ga ulanadi
2. Server task o'zgarganda broadcast qiladi
3. Client "task_updated" event oladi
4. AdminTasksProvider.fetchTasks() chaqiriladi
5. UI yangilanadi


10. PROVIDER ARXITEKTURASI
================================================================

main.dart da MultiProvider:
- AdminTasksProvider → task list, status filter, CRUD
- VideoDownloadProvider → video cache management

AdminTasksProvider methods:
- init() → WS ulash + fetchFilials + fetchTasks
- fetchTasks() → GET /api/tasks?date=...
- fetchFilials() → GET /api/filials
- updateTaskStatus(taskId, status, date)
- deleteTask(taskId)
- setSelectedDate(date) → fetch + rebuild
- toggleStatusFilter(status) → local filter
- tasksForFilial(filialId) → filtered list
